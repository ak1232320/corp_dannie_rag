# Vision (видение и объём)

## Цель

Повторить пайплайн rag-tutorial (`данные → чанки → индекс → поиск → ответ`) на
**своих** данных и довести до уровня «Отлично»: 1000+ записей/чанков, ≥2 улучшения,
тесты, документация.

## Объём (scope)

- Корпус: **1000+** записей (факт: 1200 испытаний → 8585 чанков).
- Поиск: TF-IDF (baseline) **и** гибрид (TF-IDF + семантика).
- Ответ: экстрактивный **и** LLM (OpenRouter) с фолбэком.
- UI: Streamlit (вопрос → фрагменты → ответ → источники).
- Оценка: retrieval@k / hit-rate / MRR; сравнение baseline vs гибрид.

## Критерии успеха

1. Полный воспроизводимый пайплайн из одной команды на каждый шаг.
2. 3 демо-вопроса дают ответ со ссылками; 1 негативный → корректный **отказ**.
3. Метрики качества посчитаны; гибрид ≥ baseline (и заметно лучше на
   кросс-язычных запросах и при отказе).
4. Тесты зелёные (`uv run pytest`).
5. Всё ставится в **локальное** окружение (`uv` + `.venv`).

## Не-цели

- Продакшн-развёртывание, аутентификация, масштабирование на весь датасет (60k).
- Реальная клиническая эксплуатация и медицинские выводы.
- Дообучение моделей (используем готовые предобученные эмбеддинги/LLM).

## Ключевые риски и решения

| Риск | Решение |
|---|---|
| Лексика не разделяет офтоп и домен | Отказ по семантическому порогу (улучшение 1/2) |
| Нет ключа LLM | Экстрактивный фолбэк — приложение всегда работает |
| Большой исходник (152 МБ) | Стратифицированная выборка 1200; исходник в `.gitignore` |
| Англоязычные данные, запросы на RU | Мультиязычная модель эмбеддингов |
